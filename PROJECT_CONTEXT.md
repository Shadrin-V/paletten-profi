# Project Context

## Project

**Ladungsträger Platform + Internal ERP**

This repository stores living project documentation for a lean ERP implementation and possible B2B marketplace integration.

## Goal

Replace the current Minisys-based workflow with a modern, practical ERP setup, likely based on ERPNext or a comparable standard ERP system.

The focus is not to over-document the business, but to quickly validate standard ERP processes, launch first workflows, and improve documentation during implementation.

## Business context

- Market: Germany / DACH
- Business type: trading and logistics around pallets and other Ladungsträger
- Purchases: Eastern Europe and other supplier markets
- Sales: mainly Germany / DACH
- Logistics: own trucks for some routes, external carriers for most transport
- Warehouse: own physical warehouse

## Key ERP areas

- Customers and suppliers
- Items and product categories
- Sales orders
- Purchase orders
- Pricing and margin calculation
- Warehouse and stock movements
- Logistics and dispatching
- Delivery notes and invoices
- Payments, reminders and finance-related workflows
- Employee / representative commissions

## Implementation principle

Use standard ERP functionality first. Customize only when a real business gap is proven.

Documentation should stay lean, practical and implementation-oriented.
